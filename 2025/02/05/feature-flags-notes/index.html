Notas sobre Feature Flags

El Auge y la Complejidad de los Feature Flags en DevOps

En la última década, los feature flags se han convertido en una herramienta fundamental dentro del ecosistema de despliegue, DevOps y mantenimiento, especialmente para la implementación continua. Permitir que operaciones e ingeniería operen de forma independiente ha sido una de sus mayores ventajas, pero no viene sin desafíos.

Ventajas y Desafíos:

  • Ventajas: Los feature flags permiten lanzamientos controlados, experimentación y la capacidad de activar o desactivar características sin necesidad de nuevos despliegues. Esto facilita el despliegue continuo y la innovación ágil.

  • Desafíos: Sin embargo, añaden complexity al código, aumentan el overhead de pruebas, y pueden llevar a bugs si no se gestionan adecuadamente. La lack de procesos automatizados para el seguimiento y la limpieza de feature flags puede resultar en:

  • Zombie flags: Feature flags olvidados que siguen en el código, incrementando el carrying cost y afectando la legibilidad y mantenimiento del código.

  • Unfinished business: Despliegues incompletos de feature flags, causando problemas a largo plazo.

  • Ghost flags: Feature flags eliminados pero aún manipulados por usuarios sin saberlo.

Uso Inapropiado:

  • Un feature flag pensado para un lanzamiento puede usarse para control de acceso a largo plazo, lo que puede llevar a problemas si el servicio externo falla. Esto ha llevado a algunos equipos a separar los sistemas de feature flags en:

  • Short-lived flags para lanzamientos temporales.

  • Long-lived flags para controles de acceso o características permanentes.

Esta separación, aunque lógica, trae consigo:

  • Lack of portability: Dificultad para mover un flag de un sistema a otro sin cambios de código.
  • User confusion: Frustración por recordar en qué sistema está cada flag.
  • Lack of context: Nombres de flags que no explican su propósito sin documentación adicional.

La Realidad de la Gestión de Feature Flags:

El sentido común sugiere que los feature flags deben ser temporales y limitados, pero la realidad del desarrollo de software moderno, con su ritmo rápido y naturaleza multifuncional, hace esto difícil de mantener. Además, la feature management va más allá del ámbito de DevOps, abarcando también la gestión de características para usuarios en diferentes contextos.

Propuestas para Mejorar:

  • Herramientas de Gestión de Feature Flags: Necesitamos herramientas que puedan manejar tanto short-lived como long-lived flags de manera cohesiva, con funciones como:

  • Metadatos adicionales para flags de corta duración, incluyendo fechas de expiración.

  • Asignación de propietarios para cada flag.
  • Capacidad de cambiar el propósito de un flag dentro de la misma herramienta.

  • Políticas automatizadas para la gestión y revisión de flags.

  • Automatización: Implementar procesos automatizados de auditoría y mantenimiento para evitar el crecimiento descontrolado de la complejidad.

Conclusión:

Necesitamos redefinir qué significa la feature management en la industria, alineando las prácticas y herramientas con la realidad operativa de nuestros negocios. Esto podría incluir la creación de herramientas que manejen la complejidad inherente de los feature flags, mejorando así la eficiencia y reduciendo los riesgos asociados con su gestión.
